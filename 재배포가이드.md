# 🚀 Boostar 코드 수정 후 재배포 가이드

## 📋 빠른 배포 방법

### 방법 1: PowerShell 스크립트 사용 (가장 쉬움) ⭐

```powershell
.\deploy-quick.ps1
```

스크립트가 자동으로:
1. 변경사항 확인
2. Git에 추가
3. 커밋 (메시지 입력)
4. GitHub 푸시
5. Vercel 자동 재배포 트리거

---

### 방법 2: 수동으로 명령어 실행

```powershell
# 1. 변경사항 확인
git status

# 2. 모든 변경사항 추가
git add .

# 3. 커밋 (메시지 변경)
git commit -m "수정 내용 설명"

# 4. GitHub에 푸시
git push
```

**푸시하면 Vercel이 자동으로 새 배포를 시작합니다!** 🎉

---

## 📝 커밋 메시지 예시

```powershell
git commit -m "Fix: 멤버 페이지 버그 수정"
git commit -m "Add: 새로운 기능 추가"
git commit -m "Update: UI 개선"
git commit -m "Refactor: 코드 리팩토링"
```

## 🔍 배포 상태 확인

### Vercel Dashboard에서 확인:
1. https://vercel.com/dashboard 접속
2. `boostar-dev` 프로젝트 클릭
3. "Deployments" 탭에서 진행 상황 확인

### 배포 단계:
```
⏳ Building... (빌드 중)
⏳ Deploying... (배포 중)
✅ Ready (완료!)
```

## ⚠️ 빌드 실패 시

### 오류 확인:
1. Vercel Dashboard → Deployments
2. 실패한 배포 클릭
3. "Build Logs" 확인

### 로컬에서 먼저 테스트:
```powershell
# 빌드 테스트
npm run build

# 오류가 있으면 수정 후 다시 푸시
```

## 🎯 자주 사용하는 명령어

### 특정 파일만 커밋:
```powershell
git add src/app/console/[artistCode]/projects/[projectCode]/page.tsx
git commit -m "Update: 프로젝트 페이지 수정"
git push
```

### 여러 파일 커밋:
```powershell
git add src/components/ src/app/
git commit -m "Update: 컴포넌트 및 페이지 수정"
git push
```

### 마지막 커밋 수정:
```powershell
git add .
git commit --amend -m "수정된 커밋 메시지"
git push --force
```

## 📊 배포 시간

- **빌드**: 2-3분
- **배포**: 30초-1분
- **총 소요 시간**: 약 3-4분

---

## 💡 팁

1. **작은 변경사항도 자주 배포**: 큰 변경사항보다 작은 변경사항을 자주 배포하는 것이 안전합니다
2. **커밋 메시지는 명확하게**: 나중에 변경 이력을 추적하기 쉽습니다
3. **로컬 빌드 테스트**: 푸시 전에 `npm run build`로 테스트하면 실패를 방지할 수 있습니다

---

## 🆘 문제 해결

### "nothing to commit" 오류
→ 변경사항이 없거나 이미 커밋되었습니다. `git status`로 확인하세요.

### "push rejected" 오류
→ 다른 곳에서 푸시했을 수 있습니다. `git pull` 후 다시 푸시하세요.

### 배포가 시작되지 않음
→ Vercel Dashboard에서 수동으로 "Redeploy" 클릭하세요.
